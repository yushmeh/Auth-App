# AUTH_SYS — Система аутентификации с веб-интерфейсом

Учебный проект: система регистрации и авторизации пользователей с защитой от брутфорса. Бэкенд на **Python + Flask**, фронтенд — одностраничное приложение в стиле ретро-терминала.

---

## Возможности

- Регистрация и авторизация пользователей
- Защита от брутфорса: блокировка аккаунта после 3 неверных попыток
- Визуальный обратный отсчёт блокировки прямо в интерфейсе
- Защищённая зона с мониторингом системных метрик (CPU, RAM, диск, сеть)
- Логирование всех событий безопасности в файл `app.log`
- CRT-эстетика интерфейса: сканлайны, glitch-анимация, терминальный шрифт

## REST API

Все эндпоинты принимают и возвращают JSON.

### `POST /api/register`

Регистрация нового пользователя.

**Тело запроса:**
```json
{ "login": "alice", "password": "secret" }
```

**Ответы:**

| Статус | Описание |
|---|---|
| `200` | Пользователь успешно создан |
| `400` | Логин уже занят или поля пустые |

---

### `POST /api/login`

Авторизация пользователя.

**Тело запроса:**
```json
{ "login": "alice", "password": "secret" }
```

**Ответы:**

| Статус | Описание |
|---|---|
| `200` | Вход выполнен, сессия открыта |
| `400` | Неверный пароль, в ответе `attempts_left` |
| `429` | Лимит попыток исчерпан, в ответе `locked_for` (секунд) |
| `423` | Аккаунт уже заблокирован, в ответе `locked_for` (секунд) |

---

### `POST /api/logout`

Завершение сессии.

---

### `GET /api/status` *(требует авторизации)*

Возвращает случайные системные метрики.

**Пример ответа:**
```json
{
  "ok": true,
  "cpu": 42,
  "ram": 67,
  "disk": 81,
  "uptime": 320,
  "net_in": 154,
  "net_out": 38,
  "user": "alice"
}
```

| Статус | Описание |
|---|---|
| `200` | Метрики получены |
| `401` | Сессия не активна |

## Настройка

Все параметры находятся в `config.py`:

```python
MAX_LOGIN_ATTEMPTS   = 3      # Попыток до блокировки
LOCKOUT_DURATION_SEC = 10     # Длительность блокировки в секундах

FLASK_HOST = "127.0.0.1"      # Хост сервера
FLASK_PORT = 5000             # Порт сервера
SECRET_KEY = "change-me-..."  # Секрет для Flask-сессий (смените в продакшене!)
```

## Логирование

События записываются в `app.log` в формате:

```
2025-01-15 14:32:01 - INFO    - Новый пользователь зарегистрирован: 'alice'.
2025-01-15 14:32:18 - INFO    - Успешный вход: 'alice'.
2025-01-15 14:35:44 - WARNING - Неудачная попытка входа для 'bob' (1/3).
2025-01-15 14:35:51 - WARNING - Неудачная попытка входа для 'bob' (2/3).
2025-01-15 14:35:57 - WARNING - Превышен лимит попыток для 'bob'. Блокировка на 10 с.
```

Предупреждения (`WARNING`) также выводятся в консоль при запуске сервера.

## Архитектурные решения

**Разделение слоёв.** Никакой бизнес-логики во Flask-роутах и никакого HTTP во `auth_service.py`. Роуты только транслируют результаты сервиса в HTTP-ответы.

**Блокировка без `time.sleep`.** При превышении лимита попыток сервис сохраняет `timestamp` окончания блокировки (`time.monotonic()`). Ответ возвращается клиенту мгновенно, а обратный отсчёт ведёт браузер.

**Два состояния блокировки.** `JUST_LOCKED` (лимит только что исчерпан) и `ALREADY_LOCKED` (попытка войти во время активной блокировки) — разные коды ответа (`429` и `423`), что позволяет фронтенду корректно синхронизировать таймер.

## Ограничения

- Данные пользователей хранятся в открытом виде в `users.json` — не используйте в продакшене без хеширования паролей (например, `bcrypt`).
- Состояние блокировок хранится только в памяти процесса — сбрасывается при перезапуске сервера.
- Сервер однопоточный (Flask dev mode) — для продакшена используйте `gunicorn` или `waitress`.
