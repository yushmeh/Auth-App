# AUTH_SYS — Система аутентификации с веб-интерфейсом

Cистема регистрации и авторизации пользователей с защитой от брутфорса. Бэкенд на **Python + Flask**, фронтенд — одностраничное приложение в стиле ретро-терминала.

---

## Возможности

- Регистрация и авторизация пользователей
- Защита от брутфорса: блокировка аккаунта после 3 неверных попыток
- Визуальный обратный отсчёт блокировки прямо в интерфейсе
- Защищённая зона с мониторингом системных метрик (CPU, RAM, диск, сеть)
- Логирование всех событий безопасности в файл `app.log`
- CRT-эстетика интерфейса: сканлайны, glitch-анимация, терминальный шрифт

---

## Стек технологий

| Слой | Технологии |
|---|---|
| Бэкенд | Python 3.10+, Flask 3.x |
| Хранилище | `users.json` (плоский файл) |
| Логирование | Стандартный модуль `logging` |
| Фронтенд | Vanilla HTML / CSS / JavaScript |

---

## Структура проекта

```
auth_app/
├── config.py          # Все константы: пути, лимиты, параметры Flask
├── logger_config.py   # Настройка logging (файл + консоль)
├── auth_service.py    # Бизнес-логика: регистрация, авторизация, блокировка
├── server.py          # Flask REST API
├── static/
│   └── index.html     # Весь фронтенд (HTML + CSS + JS в одном файле)
├── users.json         # База пользователей (создаётся автоматически)
└── app.log            # Лог событий (создаётся автоматически)
```

## Быстрый старт

**1. Клонируйте или скачайте проект**

```bash
cd auth_app
```

**2. Установите зависимости**

```bash
pip install flask
```

**3. Запустите сервер**

```bash
python server.py
```

**4. Откройте браузер**

```
http://127.0.0.1:5000
```

## Ограничения

- Данные пользователей хранятся в открытом виде в `users.json` — не используйте в продакшене без хеширования паролей (например, `bcrypt`).
- Состояние блокировок хранится только в памяти процесса — сбрасывается при перезапуске сервера.
- Сервер однопоточный (Flask dev mode) — для продакшена используйте `gunicorn` или `waitress`.
